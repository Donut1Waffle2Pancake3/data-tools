# SEO Rules

This document defines editorial and SEO standards for tool pages, especially FAQ and How It Works sections.

## FAQ Sections

### Core Principle

All content must prioritize real user search intent, not generic explanations.

Write for:

- people trying to solve a specific problem
- not people trying to learn a definition

### 1. Page Purpose Clarity

Every tool page must clearly answer:

- what does this tool do?
- when would someone use it?
- what problem does it solve?

Avoid vague descriptions. Be concrete and outcome-focused.

### 2. Writing Style

- Use short, direct sentences.
- Avoid filler and repetition.
- Avoid "this tool allows you to..." phrasing.
- Do not sound academic or encyclopedic.

Prefer:

- "Paste your JSON to view it as a structured tree."

Over:

- "This tool allows users to visualize JSON data..."

### 3. SEO Strategy (Critical)

Content must target high-intent queries, for example:

- "why is my JSON not working"
- "how to view large JSON file"
- "fix invalid JSON error"

Do not target generic textbook queries, for example:

- "what is JSON"
- "what is CSV"
- "what is data"

If a question sounds like a textbook heading, do not include it.

### 4. FAQ Section Rules

#### 4.1 Question Quality

Each FAQ must pass this test:

Would a frustrated user actually type this into Google?

If not, reject it.

#### 4.2 Types of Questions to Include

Prioritize:

- Problem-based:
  - Why is my JSON not showing?
  - Why is my file not working?
- How-to (specific):
  - How do I view nested JSON?
  - How do I open a large JSON file?
- Edge cases:
  - Can this handle large files?
  - What happens if my JSON is invalid?
- Trust/safety:
  - Is my data uploaded or stored?

#### 4.3 Types of Questions to Avoid

Do not include:

- Definitions:
  - What is JSON?
  - What is a CSV file?
- Generic phrasing:
  - How do I use this tool?
  - What does this tool do?
- Awkward SEO stuffing:
  - Can I use a JSON viewer without coding software?

#### 4.4 Answer Guidelines

Answers must be:

- 1-3 short paragraphs max
- immediately actionable
- free of fluff

Include:

- a clear explanation
- bullet points for common issues when relevant

Avoid:

- long intros
- repeating the question
- over-explaining basics

### 5. "How It Works" Section

Keep it simple and procedural:

1. Paste or upload your data
2. Click the main action button
3. View or download results

Do not over-explain implementation details.

### 6. Tone

- practical
- direct
- neutral

Do not:

- use hype language
- use emojis
- use marketing fluff

### 7. Keyword Usage

Include the primary keyword naturally in:

- H1
- first paragraph

Use keyword variations naturally in FAQs.

Do not:

- force keywords into every sentence
- repeat the same phrase excessively

### 8. Output Expectations for Cursor

When generating a page, Cursor must:

- avoid generic FAQs
- prefer problem-solving questions
- write answers that reflect real usage scenarios
- keep content concise and structured

If unsure, default to:

"What would someone search when this tool fails or is confusing?"

### 9. Example: Good vs Bad FAQ

Bad:

- What is JSON?
- How do I use this tool?

Good:

- Why is my JSON not displaying?
- How can I view nested JSON data?
- Can this handle large JSON files?

### 10. Final Check (Required)

Before completing a page, verify:

- no generic definition questions exist
- at least 50% of FAQs are problem-based
- answers are concise and actionable
- content reflects real user intent

If any fail, rewrite.

### Summary

This site wins by:

- being fast
- being simple
- solving specific problems

Not by:

- teaching concepts
- writing generic SEO content

## How It Works Sections

Best practices for clarity, SEO alignment, and consistency across tool pages.

### 1. Anchor It to Search Intent

The H2 should match how people search.

Good:

- How to convert JSON to CSV
- How to remove duplicate lines
- How to sort lines of text

Bad:

- "How it works"
- "Using this tool"

Rule: Always use "How to + primary keyword".

### 2. Keep It to Exactly 3 Steps

This is both UX and SEO.

Why:

- Faster to scan
- More consistent across pages
- Matches user mental model

### 3. Use Action-Based Step Titles

Each step should start with a verb.

Good:

- Paste your JSON
- Convert to CSV
- Download the file

Bad:

- Your JSON input
- Conversion process
- Output results

### 4. Focus on Outcome, Not UI

Users care about results, not buttons.

Good:

- "Download the CSV file"

Bad:

- "Click the convert button"
- "Press generate"

### 5. Keep Each Step Minimal

- Title: 3-8 words
- Description: 1 short sentence

Example:

2. Convert JSON to CSV  
The tool automatically maps objects to rows and fields to columns.

### 6. Do Not Over-Explain

Avoid:

- technical explanations
- implementation details
- edge cases (belong in FAQ)

### 7. Use a Consistent Pattern Across All Pages

Every tool should feel structurally consistent.

Core pattern:

1. Input
2. Process
3. Output

### 8. Description Above Steps Matters

Keep it:

- 1-2 sentences
- simple
- keyword aligned

Template:

Paste your [input], [perform action], and [get result]. Everything runs in your browser with no uploads.

### 9. Reinforce Key Value Props

Every How It Works section should subtly reinforce:

- instant
- no uploads
- browser-based
- simple

### 10. Avoid Redundant Steps

Bad:

1. Paste your data
2. Review your data
3. Process your data
4. Download

Too many steps, low value.

### 11. Do Not Split Options Into Steps

Options are not steps.

Bad:

2. Choose delimiter  
3. Choose header option  
4. Generate output

Good:

2. Choose your options

### 12. Keep Language Predictable

Users should recognize the same pattern site-wide.

Reusable phrasing:

- Paste your [input]
- Choose options
- Download or copy the result

### 13. Do Not Mention Internal Logic

Avoid:

- "the algorithm"
- "this function"
- "parsing logic"

Keep copy user-focused.

### 14. Avoid Conditional Complexity

Do not say:

- "If X, then Y"
- "Depending on your data..."

That belongs in FAQ, not How It Works.

### 15. Optimize for Scannability

Each step should be readable in under 2 seconds.

## High-Quality How It Works Template (Reusable)

H2:

How to [primary keyword]

Description:

Paste your [input], [action], and [result]. Everything runs in your browser with no uploads or data leaving your device.

Steps:

1. Paste your [input]  
   Short explanation.
2. [Perform action]  
   Short explanation.
3. Download or copy the result  
   Short explanation.

## Example (Strong)

How to convert JSON to CSV

Paste your JSON, convert it to CSV, and download the result instantly. Everything runs in your browser with no uploads or data leaving your device.

1. Paste or upload your JSON  
   Paste JSON data into the input or upload a file.
2. Convert JSON to CSV  
   The tool automatically maps objects to rows and fields to columns.
3. Download the CSV file  
   Download the converted file or copy it to your clipboard.

## What Actually Matters Most

If you only focus on a few things:

- H2 matches search intent
- Exactly 3 steps
- Action-based wording
- No UI noise (for example, "click button")
- Consistent structure across pages

That is where most of the value comes from.

## Header and Page Title Best Practices

Practical best practices that impact rankings and CTR for hero/header sections and page titles.

### 1. H1 (On-Page Title) Best Practices

#### Match the Primary Query Exactly

Your H1 should closely mirror what users search.

Good:

- Convert JSON to CSV
- Remove Duplicate Lines
- Sort Lines

Bad:

- Easily Convert JSON Files into CSV Format Online
- The Best Tool to Remove Duplicate Lines

Rule:
Keep it literal, short, and intent-matching.

#### Keep It Short

- Ideal: 3-6 words
- Max: ~8 words

Why:

- Easier to scan
- Stronger keyword clarity
- Less dilution

#### Avoid Modifiers

Do NOT add:

- best
- free
- online
- fast

Those belong in the meta title, not the H1.

#### Use Verb-Based Structure

Most tools should follow:

Verb + object

Examples:

- Convert CSV to JSON
- Trim Whitespace
- Filter CSV Rows

### 2. Page Title (Meta Title) Best Practices

This is what shows in Google. It is different from H1.

#### Include Keyword + Modifier

Structure:

[Primary keyword] - [modifier]

Examples:

- Convert JSON to CSV - Free Online Tool
- Remove Duplicate Lines - Fast Text Cleaner
- CSV Column Remover - Remove Columns Online

#### Use High-CTR Modifiers

Best performing:

- Online
- Free
- Tool
- Converter
- Viewer

#### Keep Length Tight

- Ideal: 50-60 characters
- Avoid truncation

#### Put Keyword First

Good:

- Convert JSON to CSV - Free Tool

Bad:

- Free Tool to Convert JSON to CSV

### 3. Hero Subheading (Under H1)

This impacts both SEO and UX.

#### Structure

- 1-2 sentences
- Reinforce:
  - what it does
  - speed
  - no uploads

#### Strong Template

Paste your [input], [perform action], and [get result]. Everything runs in your browser with no uploads.

#### Example

Paste your JSON, convert it to CSV, and download the result instantly. Everything runs in your browser with no uploads.

#### Why This Works

- Reinforces keyword
- Improves clarity
- Builds trust (no uploads)

### 4. Header Section (Hero) Layout Best Practices

Current structure is good. Keep it consistent:

- H1
- Subheading
- Tool UI

#### Do Not Add Noise

Avoid:

- long intros
- multiple paragraphs
- feature lists

Users came to use the tool.

### 5. Keyword Placement Strategy

Where to include primary keyword:

- H1 -> exact or near-exact
- Subheading -> once
- H2 ("How to...") -> once
- FAQ -> 1-2 variations

#### Avoid Overuse

Do NOT:

- repeat exact phrase 5+ times
- force it into every sentence

### 6. Consistency Across Pages

Every page should follow the same pattern:

- H1 = literal tool name
- Meta title = keyword + modifier
- Subheading = outcome + trust

This builds:

- UX familiarity
- SEO clarity
- scalable system

### 7. Avoid These Common Mistakes

- Keyword stuffing:
  - "Convert JSON to CSV Online Free Tool Converter"
- Mismatch between H1 and title:
  - H1: "JSON Tool"
  - Title: "Convert JSON to CSV"
- Overly clever naming:
  - "Data Transformer Pro"
- Long intros before tool:
  - Users bounce fast

### 8. Ideal Example (Full Header)

H1:

Convert JSON to CSV

Meta Title:

Convert JSON to CSV - Free Online Tool

Subheading:

Paste your JSON, convert it to CSV, and download the result instantly. Everything runs in your browser with no uploads.

### 9. What Actually Moves the Needle

If you focus on only a few things:

- H1 matches exact search intent
- Meta title includes keyword + modifier
- Subheading clearly explains outcome
- No fluff in hero section
- Consistency across all pages

That is where most of the SEO + UX impact comes from.
