# site_rules.md

## Purpose

Defines **structure + layout** for all tool pages.  
Content = `seo_rules.md`.

---

## 1. Page Template (MANDATORY)

Every page follows this order:

### 1. Hero

- H1 (primary keyword)  
- Subheading (1–2 sentences)  
- Tool UI (input + output)

### 2. How To

- Eyebrow: HOW IT WORKS  
- H2: "How to [keyword]"  
- Description (1–2 sentences)  
- **Exactly 3 steps**  
  - Verb title  
  - 1 short sentence  

### 3. Why Use

- Eyebrow: WHY USE THIS TOOL  
- H2  
- Intro (1–2 sentences)  
- 3–4 benefits:  
  - 3–5 word title  
  - 1 sentence  

### 4. Privacy

- Eyebrow: PRIVACY  
- H2  
- Intro (1–2 sentences)  
- Must state:  
  - local (on-device)  
  - no uploads/storage  
  - browser-based  

### 5. FAQ

- Eyebrow: FAQ  
- H2  
- 5–7 questions  
- Answers: 1–3 sentences  

### 6. Related Tools

- Header  
- 3–6 internal links  

---

## 2. Structural Rules (STRICT)

- Order is fixed  
- All 6 sections required  
- No new/removal/reordering  
- Match existing layout/components  

---

## 3. Tool UI

- Directly under hero  
- Instant/near-instant  
- Prefer auto-detect  
- Minimize actions  

Avoid:

- multi-step flows  
- unnecessary buttons  

---

## 4. Scope

This file: **structure/layout**  
`seo_rules.md`: **content/tone/keywords**

---

## 5. Internal Links

- 3–6 related tools  
- Prefer same category  
- Clear tool-name anchors  

---

## 6. Naming + Slugs

**H1**

- Short, literal, intent match  

**Slug**

- lowercase  
- hyphenated  
- match tool  

Examples:

- `json-to-csv`  
- `remove-duplicate-lines`  

---

## 7. Global Changes

Allowed if:

- improves multiple pages  
- increases consistency  
- reuses components  

Must:

- keep current design  
- not break pages  

Avoid:

- redesigns  
- one-off styles  
- unnecessary refactors  

---

## 8. New Pages

Must:

- include all 6 sections  
- match structure/layout  
- include related tools  

Do not:

- skip/add sections  
- change structure  

---

## 9. Editing Pages

Must:

- preserve structure  
- improve within sections  

Avoid:

- removing/reordering  
- unnecessary rewrites  

---

## 10. Done Criteria

- All 6 sections present  
- Correct order  
- How To = 3 steps  
- FAQ = 5–7  
- Related tools included  
- Matches site layout  

---

## 11. Execution (Cursor)

- Structure → `site_rules.md`  
- Content → `seo_rules.md`  
- No new patterns  
- Keep scope tight  
- One page per run
